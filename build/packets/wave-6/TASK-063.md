## Notes

- Use `seed_post_curation` from `tests.fixtures` as the base for most DB tests. The fixture has 4 color tokens and 1 spacing token.
- For composite assembly tests, use the helper functions to add typography and shadow atomic tokens to the seeded DB.
- The multi-mode tests require manually inserting a second mode ("Dark") with token_values. The fixture only provides a single "Default" mode.
- DTCG tests should parse the generated JSON back to a dict and assert on structure, not string content (to avoid whitespace sensitivity).
- CSS alias tests need to create an alias token first (via direct INSERT), then verify the generated CSS uses `var()` syntax.
- Use `@pytest.mark.timeout(10)` on DB-heavy tests as safety.
