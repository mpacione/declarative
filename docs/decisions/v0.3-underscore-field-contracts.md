# v0.3 Underscore-Field Contracts

**Status:** Descriptive documentation, 2026-04-18.
**Related:** Priority 1A per `docs/continuation-v0.3-next-session.md` §4; **resolved via bridge** per `docs/decisions/v0.3-canonical-ir.md`.

## Summary

Five IR fields carry load-bearing state with a leading underscore. They round-trip through dd markup as ordinary properties (verified in the Priority 0 probe — no grammar-first-class treatment needed). This doc captures what each field *is* and who reads/writes it, so a future reader understands the semantics without having to grep.

All five are per-element (on an element dict) except `_node_id_map` which is top-level on the spec.

## `_node_id_map` (top-level, not per-element)

| | |
|---|---|
| **Type** | `dict[str, int]` — maps element-id (eid) → DB node_id |
| **Set by** | `dd/ir.py::build_composition_spec:1388` (extract path — real DB ids); `dd/compose.py:316,1014` (synthesis path — synthetic negative ids) |
| **Read by** | `dd/renderers/figma.py` 11 sites, `dd/compose.py:1028`, `dd/ir.py:1016,1097` |
| **Semantics** | Bridge between the IR's stable eid identifiers and the underlying Figma/DB node_id. Used for override resolution, text substitution, component-instance wiring. |
| **Lifecycle** | Populated at spec construction; never mutated after. Serialized losslessly via dd markup as `_node_id_map { map "<eid>" <nid> ... }`. |

**Why underscored:** it's an in-memory index, not a Figma-visible property. The underscore flags "implementation detail of the serde, not part of the design intent."

## `_mode1_eligible` (per-element)

| | |
|---|---|
| **Type** | `bool` |
| **Set by** | `dd/ir.py::map_node_to_element:342` |
| **Read by** | `dd/verify_figma.py:182` (gate for `KIND_TYPE_SUBSTITUTION` error reporting) |
| **Semantics** | `True` when the DB has a component_key / component_figma_id / valid INSTANCE ref that would let Mode 1 `createInstance()` succeed. `False` for name-only classified frames. |
| **Lifecycle** | Set once during extract; read once during verify. |

**Why underscored:** it's metadata about *what the renderer was allowed to do* — influences error classification but isn't itself a design property. ADR-007 Defect-1 class depends on this signal to avoid false positives (name-only classified FRAMEs are correctly rendered as frames; only those with real INSTANCE capability should flag "should be instance").

## `_corpus_source_node_id` (per-element)

| | |
|---|---|
| **Type** | `int` (DB node_id of the donor) |
| **Set by** | `dd/composition/providers/corpus_retrieval.py:434`; propagated via `dd/compose.py:160` |
| **Read by** | Test fixtures only (none in dd/ production code reads it as of 2026-04-18) |
| **Semantics** | v0.2 provenance marker — records which DB corpus node was the source of a retrieved subtree. Not currently consumed by renderer or verifier. |
| **Lifecycle** | Set by corpus_retrieval provider; persists through compose; silently dropped by renderer. |

**Why underscored:** it's provenance, not design intent. Invariant-7 technically flags this as a raw int value — per Priority 0 conservative, this is acceptable as a bridge-field, not a grammar concern. If CRAG SCREEN_EDIT mode (v0.3 §5.1) productizes provenance, this field becomes part of the `retrieved` provenance annotation per architecture §2.6 rather than an underscore field.

## `_original_name` (per-element)

| | |
|---|---|
| **Type** | `str` |
| **Set by** | `dd/ir.py:1271` (per element from Figma node name); `dd/ir.py:1378` (screen-level from screen name) |
| **Read by** | Renderer for placeholder labels (e.g. missing-component wireframes); debugging; structured error context |
| **Semantics** | The original Figma layer name (e.g. `"iPhone 13 Pro Max - 8"`, `"button/toolbar"`). Purely informational; not a design property. |
| **Lifecycle** | Set at extract; read at render/verify for diagnostics. |

**Why underscored:** it's source-system metadata carried for debugging/diagnostic use. The renderer DOES use it for placeholder labels, which makes it quasi-functional, but the function is "make this intermediate state legible to a human reader," not "express design intent."

## `_composition` (per-element)

| | |
|---|---|
| **Type** | `list[dict]` — each entry is a catalog children-composition slot spec |
| **Set by** | `dd/compose.py:961` (from `dd/templates.py::extract_child_composition`) |
| **Read by** | `dd/renderers/figma.py:1376,1379` (`_emit_composition_children`) |
| **Semantics** | Tells the renderer to emit specific child instance types at specific positions in a template component. E.g., a toolbar template's `_composition` lists which button instances to create in which order. |
| **Lifecycle** | Set by compose_screen when a catalog template has `children_composition`; read by renderer during Phase 1/2 node creation. |

**Why underscored:** it's a renderer directive derived from the catalog, not something the LLM or designer would author. The final IR *emitted to Figma* has the composition applied (real child elements); the `_composition` field is the instruction used to produce those children. The underscore flags "intermediate state in the pipeline, not the end state."

Legend: ● = reads-or-writes this field in meaningful way.

## Non-invariants

- The LLM does not emit any underscore field. They are all pipeline-internal.
- A user authoring dd markup directly could technically include them (syntax permits it) but they'd be ignored or overwritten by the pipeline.
- The dd markup serde preserves them losslessly; this is convenient for in-memory round-trips but does NOT mean they're part of the *public* grammar.

## When to revisit

If a feature needs one of these fields to be author-visible (e.g. design-intent provenance annotation `(retrieved src="donor:142" conf=0.91)` per architecture §2.6), the field graduates from underscore-pipeline-metadata to first-class grammar. The graduation is a focused PR: add grammar syntax + drop the underscore + update callers.

Until that graduation happens, underscore = "internal; serde preserves but don't read in LLM prompts or author in markup by hand."
