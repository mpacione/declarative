# v0.1.5 Plan — Mode 3 generator uplift

> **Status:** converged decision memo, 2026-04-17 (pt 8).
> **Source streams:** [`screen-archetypes.md`](./screen-archetypes.md) (α) · [`generation-density-design.md`](./generation-density-design.md) (β) · [`evaluation-rubric-calibration.md`](./evaluation-rubric-calibration.md) (γ).
> **Informs:** ADR-008 follow-on execution for the v0.1.5 sprint.
> **Precedent:** same pattern as the Mode-3 ADR-008 trilayer (α/β/γ research → converged memo → implementation).

## 1. Executive summary

Mode-3 v3 delivers the pipeline: the renderer is fine, the provider architecture holds, 204/204 round-trip parity persists, and 4 of 12 canonical prompts reach VLM-ok + 5 reach VLM-partial.

The three streams independently arrive at the same diagnosis:

1. **The bottleneck is the prompt/contract — not the evaluator, not the architecture.** Stream α surveyed the leading LLM-UI tools and found every strong system does plan-then-fill with a two-level IR; our single-shot flat generation is the outlier. Stream β's empirical dig into the 48 existing outputs found actual generation density is fine on specific prompts but falls apart on under-specified ones — and that the sparsity narrative only applies to 5 of 12 prompts. Stream γ confirmed the evaluation gate works for its built purpose; fixing the gate before fixing the generator is wasted effort.
2. **v0.3 critic-loop is the wrong next move.** α cites GameUIAgent's quality-ceiling result (Pearson r = −0.96 between reflection benefit and base quality — refinement plateaus below a threshold). γ cites cross-VLM ICC = 0.021 (second VLM adds ~zero signal at triple cost). Fix the generator first.
3. **The single highest-impact v0.1.5 lever is an archetype-conditioned few-shot pack.** α recommends it (A1); β's hypothesis space includes it (S2/S3); γ's open-question #8 proposes the density metric that would measure its effect. UICrit's 55% few-shot gain on UI feedback tasks is the empirical anchor.

This memo integrates the three streams into a **two-week v0.1.5 plan** with a small side-fix for the clarification-refusal pipeline bug. No ADR changes — this is ADR-008 execution, not redesign.

## 2. Per-stream headline findings

### α (screen archetypes)
- **Plan-then-fill with two-level IR** is consensus across strong academic systems (SpecifyUI, GameUIAgent, DesignCoder). DesignCoder reports +30% TreeBLEU / +29% container-match over flat baselines.
- **v0 doesn't plan** — retrieval over hand-curated exemplars + composite model + autofixer (64.7% → 93.9% error-free).
- **Figma First Draft** uses a closed menu of 4 pre-built libraries. Maps onto our provider architecture.
- **Recommendation:** A1 archetype-conditioned few-shot pack (ship) + A2 plan-then-fill (stretch behind flag). Defer B retrieval and C critic loop.
- **Expected uplift:** 4 → 7–8 VLM-ok from A1 alone; 4 → 9–10 from A1+A2.

### β (generation density)
- **The sparsity narrative is mostly wrong.** 7 of 12 prompts already emit ≥20 nodes with depth ≥2 in ≥3 of 4 runs. 05-paywall hit 48 nodes twice. Real sparsity bucket: 01/02/08/09/10.
- **12-round-trip-test is a pipeline bug, not a generation bug.** Haiku correctly asked for clarification; `extract_json` swallowed it as `[]`; pipeline treated as render failure. The 00f "regression" is actually categorically-better generator behaviour mishandled downstream.
- **max_tokens is not the bottleneck.** Max observed output 743 tokens on 2048 budget.
- **Row-shape drift on tables.** Same prompt → three different table shapes; container contract is ambiguous about "row cells."
- **Warnings are render-layer, not LLM.** 33/33 warnings on 04-dashboard are `Type 'X' has no template` — no prompt tuning fixes that.
- **Recommendation:** 240-call matrix (3 temperatures × 5 SYSTEM_PROMPT contracts × 12 prompts × 1 sample + 60-sample variance slice). ~$0.70 Haiku + optional $0.96 VLM confirmation. Predicted winner: S2 (min-count + clarify-as-empty) at T=0.3.

### γ (evaluation rubric)
- **Score distribution is bimodal.** 48.6% in 1-3, 28.6% in 7-10, only 22.9% mid-range. VLM is using 1-10 as a 2-bucket classifier.
- **Rule vs VLM disagreement: 69% (25/36).** They measure orthogonal things (nodes vs renders). Keep both.
- **Within-VLM instability documented.** 09-drawer-nav scored ok(7) in 00e and partial(5) in 00f on identical screens. ≥1-2 points of VLM noise.
- **Cross-VLM ICC = 0.021** (GameUIAgent's finding). Second critic would add ~zero signal.
- **Recommendation:** Hold the current 1-10 gate through v0.1.5. Run two cheap parallel experiments: (a) DIY human calibration on 48 existing artefacts, Spearman ρ; (b) shadow 5-dimension rubric via prompt change only. Graduate to 0-5 + anchor paragraphs in v0.2 if calibration data supports.

## 4. The v0.1.5 plan

### Week 1 — Contract + archetype (primary uplift)

**1. Pipeline-contract side-fix (30-line PR, 1h).**
- `dd/prompt_parser.py::extract_json` — when the response has >100 chars of non-JSON prose but no JSON array found, return a sentinel (e.g. `{"_clarification_refusal": response_text}`) instead of `[]`.
- Caller surfaces as `notes.md` entry + skips rendering; emits a `KIND_PROMPT_UNDERSPECIFIED` structured error (already reserved in ADR-008 failure vocabulary but not yet consumed).
- Restores 12-round-trip-test from a silent render-failure to a proper "LLM asked for clarification" artefact.

**2. Archetype library + `ArchetypeLibraryProvider` (~1 week).**
- Hand-author 8–12 canonical skeleton JSONs at `dd/archetype_library/*.json` — feed, dashboard, paywall, login, settings, search, onboarding-carousel, chat, empty-state, profile, checkout, detail. Each is a 2–3 level IR-shaped tree with 10–25 nodes.
- Lightweight classifier: literal-keyword match → Haiku 4.5 classification fallback → top-1 (and top-2 as stretch multi-example).
- New `dd/composition/providers/archetype_library.py` at priority 75 (above `UniversalCatalogProvider` 10, below `ProjectCKRProvider` 100) — project CKR still wins on instance-level; archetypes supply screen-level structure.
- `SYSTEM_PROMPT` injects the matched skeleton(s) inline with framing "inspiration, not template — modify for the specific prompt."
- Deterministic post-pass (GameUIAgent's "rarity enhancement" pattern): walk the LLM output; if a container type has < N children per its archetype metadata, pad from the skeleton's defaults.
- ~15 new unit tests; no round-trip parity regression expected (renderer untouched).

**3. SYSTEM_PROMPT S2 edit (min-count + clarify-as-empty, ~30 lines).**
- Add: "If the prompt implies a list/feed/table/carousel, emit ≥ 4 child items."
- Add: "If the prompt references an image/screen ID you can't see, emit `[]` — do NOT invent."
- Set `temperature=0.3` in `parse_prompt` (Haiku messages.create).

**4. Run β's 240-call matrix (~$1.70, 5 min parallel).**
- Confirms temperature + contract choice empirically.
- Structural measures only (no VLM during matrix — deferred to 60-call confirmation on the winning cell).
- If the prediction lands (S2 at T=0.3 wins), ship it as default. If it doesn't, the matrix still tells us where the next engineering dollar goes.

**5. Structural-density metric in `dd inspect-experiment` (~2h).**
- Per-prompt `(nodes, max_depth, container_coverage)` triple in the sanity report.
- Cheap observability that's independent of VLM judgement — catches regressions the VLM would miss.

### Week 2 — Plan-then-fill + calibration prototype (stretch)

**6. Plan-then-fill (α's A2) behind `DD_ENABLE_PLAN_THEN_FILL` flag (~1 week).**
- `dd/composition/plan.py` with two prompt templates + JSON-mode Claude calls.
- Plan LLM returns `{type, id, children: [{type, id, count_hint}]}` trees (pruned subset of our IR).
- Skeleton-grammar validator reusing `dd.composition.slots.validate_slot_child` + capability gate.
- Fill LLM takes the validated plan pinned in context; fills leaf text/icons/variants.
- Plan-diff post-hoc: if fill dropped any planned nodes, retry once with pinned plan.
- Exposes `KIND_PLAN_INVALID` on unrecoverable errors (ADR-008 failure vocabulary extension).
- Feature flag off by default; CI still uses single-call mode.
- Flag on for the 00g experiment to measure the additional uplift over A1.

**7. DIY human calibration (γ's recommendation, ~1h).**
- Matt rates each of 48 existing screenshots 0-5 on Layout / Completeness / Readability.
- Compute Spearman ρ vs Gemini's 1-10 scores.
- Deliverable: `docs/research/calibration-48.md` with the correlation, decision on whether to promote 5-dim rubric in v0.2.
- Parallel, not blocking.

**8. Shadow 5-dim rubric prompt (γ's parallel experiment, ~2h).**
- `dd/visual_inspect.py` gets an opt-in second prompt emitting the GameUIAgent 5-dim vector alongside the 1-10 score.
- Store both in sanity_report.json; gate still uses 1-10 through v0.1.5.
- Graduate to 5-dim in v0.2 if calibration shows it separates partial-vs-ok cleanly.

### Deferred to v0.2 (with reasoning)

- **A2 if not shipped in week 2** — clear next priority.
- **Corpus retrieval (α's B)** — needs either a 2nd project or a ≥1k-screen corpus; our 204-screen Dank doesn't meet the bar.
- **N-variant + VLM-pick (α's D)** — triples token cost for modest gain; deferred unless A1+A2 leaves ≥3 VLM-partials.
- **5-dim rubric graduation** — gated on calibration data from week 2's parallel exp.
- **Archetype taxonomy mined from corpus** — v0.2 investment once A1 is proven; initial 8-12 hand-authored skeletons are the MVP.

### Deferred to v0.3 (with reasoning)

- **RenderVerifier critic loop (α's C)** — per ADR-008 §phasing, gated explicitly on "base VLM-ok ≥ 8/12" per GameUIAgent's quality-ceiling result. Don't build before that floor is cleared.

### Not in plan (with reasoning)

- **Second VLM critic** — γ cited cross-VLM ICC = 0.021 (near-zero inter-rater reliability between GPT-4o / Claude / Gemini on UI). Triples cost for ~zero signal. One strict critic is sufficient.
- **0-100 absolute scale** — worse than 0-10 per arxiv 2601.03444 (ICC 0.805 vs 0.853 for 0-5). If we upgrade the scale, target 0-5 with anchor paragraphs.
- **Removing the rule-based gate** — γ showed 69% rule-vs-VLM disagreement reflects orthogonal measurement (nodes vs renders), not redundancy. Keep both.

## 6. Risk and rollback

**Rollback surface** — every v0.1.5 change is additive + behind a flag:

- **Archetype provider**: `DD_DISABLE_ARCHETYPE_LIBRARY=1` falls back to current SYSTEM_PROMPT only; universal catalog provider unchanged.
- **Plan-then-fill**: `DD_ENABLE_PLAN_THEN_FILL` off by default — no behavioural change in CI.
- **Temperature change**: single env var / call-site flip.
- **Clarification-refusal fix**: structural-error surfaced through existing ADR-006/007 channel; downstream consumers that don't read it degrade gracefully.
- **Round-trip parity**: none of these touch the renderer or extractor; 204/204 should hold.

**Principal risks:**

1. **Archetype overfit.** The LLM might copy skeleton verbatim. Mitigation: (a) frame as "inspiration, not template," (b) post-pass diff — if top-level types match the skeleton 1:1 with zero variation, retry with "diverge from the skeleton" instruction.
2. **Classifier mis-routing.** "A search page with a feed of results" → classified as feed, not search. Mitigation: top-2 skeletons as multi-example; if both misclassify, ratio of top-2 scores is close, degrade to no-archetype SYSTEM_PROMPT.
3. **Plan-then-fill drift.** Fill drops nodes. Mitigation: plan-diff + single retry with pinned plan; exposure through `KIND_PLAN_INVALID`.
4. **Matrix confirms no contract variant helps.** The research priors suggest this is unlikely but possible. If matrix comes back flat, ship A1 anyway (α-backed independent of β's matrix), and escalate the render-template gap as the next bottleneck.

## 7. What this plan doesn't attempt

- **No ADR-008 revisions.** The trilayer architecture holds. This is execution.
- **No IR schema changes.** Everything is provider-layer + prompt-layer.
- **No IngestedSystemProvider population.** shadcn templates stay empty; the archetype library provides the structural content.
- **No real variant clustering** in `dd/cluster_variants.py`. Placeholder rows stand. Real clustering is v0.2 on a classified corpus.
- **No critic loop.** GameUIAgent's ceiling effect is the explicit reason.

### Per-stream actual vs predicted

- **Stream β (matrix)** — **predicted** S2 @ T=0.3 as dominant cell; **actual** no variant beat S0 at ≥1 std-dev on ≥3 of 5 measures on ≥9 of 12 prompts. **Routed to** keep S0 + land T=0.3 (already in `3796058`). Matrix still bought us: quantified variance floor (`total_node_count` σ=3.44), confirmation that the current enriched SYSTEM_PROMPT is worth +11 nodes vs a bare-catalog control (S4), and validation of the clarification-refusal side-fix (S2 fires at 2-3/12).
- **Stream α (archetypes)** — **predicted** 7-8 VLM-ok from A1 alone; **actual** 6 VLM-ok. Near-prediction but one short. Where archetypes worked: paywall / search / round-trip-test all moved into VLM-ok via explicit skeleton injection. Where they didn't: dashboard regressed on a render-template issue (tables-as-text-stacks — not an archetype problem); vague regressed on classifier=None and T=0.3 losing the T=1.0 luck 00f had.
- **Stream γ (rubric)** — **predicted** cross-VLM ICC=0.021 would kill a second critic; **actual** we didn't need a second critic to hit 6/12. Within-VLM noise proved real: first run had 4/12 transient Gemini errors; three reruns stabilized to 6 ok / 4 partial / 2 broken. **Kept** the single-critic 1-10 gate as designed.

### Surprises worth memorializing

1. **`screen_component_instances` is empty** in the current Dank DB snapshot; corpus-mining as originally planned requires a classifier re-run (deferred to v0.2). Week 1 shipped 12 hand-authored skeletons instead, declared via `dd/archetype_library/provenance.json`.
2. **`PROXY_EXECUTE_RESULT` parse-depth bug** — the `__ok` flag is nested at `.result.result.__ok`, not top-level. 00f hit this (required `run_walks_and_finalize.py` patch); 00g hit it too (fixed in `run_render_walk.py`'s `_parse_run_result`). See `feedback_proxy_execute_parse_depth.md`.
3. **Gemini 3.1 Pro is transient-flaky** on batch runs — 4/12 VLM calls failed the first pass with 503 / timeouts; 3 reruns converged. Built-in retry (2× w/ backoff) is insufficient; the gate needs outer rerun tolerance. See `feedback_vlm_transient_retries.md`.
4. **A1 helps on structure-bearing prompts, hurts on vague ones.** When the classifier routes correctly AND the skeleton has more structure than Haiku emits by default, A1 wins (paywall / search). When classifier=None, SYSTEM_PROMPT fallback at T=0.3 is MORE conservative than 00f's T=1.0 was (vague regressed).
